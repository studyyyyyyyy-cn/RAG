# 混合检索功能使用指南

## 问题描述

之前搜索"智能吸顶灯"时无法找到正确结果，原因是：
1. 系统只使用密集向量（Dense）进行语义搜索
2. 没有使用稀疏向量（Sparse）进行关键词匹配
3. "智能吸顶灯"的语义向量与其他设备意外相似

## 解决方案

已实现完整的混合检索（Dense + Sparse + RRF融合），显著提升检索准确性。

## 使用步骤

### 1. 清理旧数据

旧的向量数据不包含稀疏向量，需要删除：

```bash
rm backend/data/vectors.json
```

### 2. 重新分块文档

在前端文档管理页面：
1. 找到需要重新索引的文档（如table.csv）
2. 点击"重新分块"按钮
3. 选择合适的分块方式（table.csv使用"表格分块"）
4. 等待分块完成

系统会自动：
- 使用BGE-M3模型生成dense + sparse向量
- 将向量存储到InMemoryVectorStore
- 支持混合检索

### 3. 测试检索

在测试问答页面：
1. 输入查询："智能吸顶灯"
2. 查看检索结果
3. 应该能看到"智能吸顶灯"排在第一位

## 技术说明

### 混合检索原理

1. **Dense向量**：捕捉语义相似度
   - 使用BGE-M3模型生成1024维向量
   - 适合理解"智能照明设备"等语义概念

2. **Sparse向量**：捕捉关键词匹配
   - 使用BGE-M3的lexical_weights
   - 适合精确匹配"吸顶灯"等关键词

3. **RRF融合**：综合两种检索优势
   - 公式：score = 1/(k+rank_dense) + 1/(k+rank_sparse)
   - k=60（标准RRF常数）

### 自动切换机制

系统会自动选择最佳存储方案：
- Milvus Lite可用 → 使用Milvus（未来支持混合检索）
- Milvus Lite不可用 → 使用InMemoryVectorStore（已支持混合检索）

## 常见问题

### Q1: 为什么需要重新分块？

A: 旧的向量数据只包含dense向量，没有sparse向量。重新分块后会生成完整的dense + sparse向量。

### Q2: 重新分块会影响现有数据吗？

A: 不会。重新分块只会更新向量索引，不会修改原始文档和分块内容。

### Q3: 如何确认系统使用了BGE-M3？

A: 查看后端启动日志，应该看到"Loading BGE-M3 model"。如果看到"Using lightweight embedder"，说明配置有误。

### Q4: 混合检索会影响性能吗？

A: 轻微影响。混合检索需要执行两次检索（dense + sparse），但通过RRF融合后效果显著提升，值得这点性能开销。
