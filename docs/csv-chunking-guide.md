# CSV 文件分块说明

## 文件：QA.csv

### 文件结构

```csv
QZST183,曙光
QZST184,九溪
QZST185,上部
QZST186,罗田
QZST187,横林
QZST188,刘厝
QZST189,坝头
QZST116,安平桥
```

**说明**：
- 第一行被识别为表头：`QZST183` 和 `曙光`
- 实际数据行：7 行
- 每行 2 个字段（编号和名称）

---

## 不同分块方式的效果

### 1. 问答分块（qa）

**分块数量**：14 个

**分块逻辑**：
- 每个键值对作为一个分块
- 每行有 2 个键值对，所以 7 行 = 14 个分块

**分块示例**：
```
Chunk 0: QZST183: QZST184
Chunk 1: 曙光: 九溪
Chunk 2: QZST183: QZST185
Chunk 3: 曙光: 上部
...
```

**适用场景**：
- ❌ 不适合这种数据
- 每个分块太小，缺乏上下文
- 键值对被拆分，失去关联

---

### 2. 通用分块（general）

**分块数量**：1 个

**分块逻辑**：
- 固定大小分块（512 tokens）
- 整个文件内容很小，只生成 1 个分块

**分块示例**：
```
Chunk 0:
QZST183 : QZST184 ; 曙光 : 九溪
QZST183 : QZST185 ; 曙光 : 上部
...
```

**适用场景**：
- ✅ 适合小文件
- 保持所有数据在一起
- 但无法单独检索某一行

---

### 3. 表格分块（table）

**分块数量**：1 个

**分块逻辑**：
- 识别 Markdown 表格格式
- CSV 转换后不是 Markdown 表格，所以整个作为一块

**适用场景**：
- ❌ 不适合（需要 Markdown 表格格式）

---

### 4. 递归分块（recursive）

**分块数量**：1 个

**分块逻辑**：
- 按分隔符递归切分
- 文件太小，不需要切分

**适用场景**：
- ✅ 适合小文件
- 保持完整性

---

### 5. 智能分块（intelligent）

**分块数量**：1 个

**分块逻辑**：
- 识别文档结构（标题、章节）
- CSV 数据没有明显结构，作为一块

**适用场景**：
- ❌ 不适合（需要有标题结构）

---

### 6. 父子分块（parent_child）

**分块数量**：
- 父分块：1 个
- 子分块：1 个

**分块逻辑**：
- 先生成父分块（1536 tokens）
- 再切分为子分块（512 tokens）
- 文件太小，只生成 1 个子分块

**适用场景**：
- ✅ 适合，但对小文件意义不大

---

## 推荐方案

### 方案 1：改进 CSV 格式（推荐）

将 CSV 改为更适合问答的格式：

```csv
问题,答案
QZST183是什么地方,曙光
QZST184是什么地方,九溪
QZST185是什么地方,上部
...
```

然后使用**问答分块**，每行作为一个问答对。

---

### 方案 2：使用通用分块

对于这种小型数据文件，使用**通用分块**或**递归分块**即可：
- 整个文件作为一个分块
- 检索时能找到所有数据
- 适合数据量小的情况

---

### 方案 3：改进 CSV 解析器

修改 CSV 解析器，让每行数据作为一个独立的文本块：

**当前格式**：
```
QZST183 : QZST184 ; 曙光 : 九溪
```

**改进后格式**：
```
编号: QZST184, 名称: 九溪
编号: QZST185, 名称: 上部
...
```

这样每行是一个完整的记录，更适合检索。

## 实施建议

### 短期方案（立即可用）

使用**通用分块**或**递归分块**：
1. 删除当前文档
2. 重新上传
3. 选择"通用分块"或"递归分块"
4. 执行分块

### 长期方案（需要开发）

改进 CSV 解析器，支持按行分块：

```python
# 修改 _parse_csv 方法
def _parse_csv(self, file_path: str) -> list[DocumentPage]:
    # ... 读取 CSV ...

    # 每行作为一个 page
    pages = []
    for i, row in enumerate(rows):
        row_text = ", ".join(f"{k}: {v}" for k, v in row.items() if v)
        pages.append(DocumentPage(
            text=row_text,
            page_number=i + 1,
            section_title=f"Row {i + 1}",
            metadata={"source": Path(file_path).name, "row_index": i},
        ))

    return pages
```

这样每行 CSV 数据会生成一个独立的分块。

## 总结

对于当前的 QA.csv 文件：

1. **不推荐使用问答分块**
   - 会生成 14 个很小的分块
   - 每个分块缺乏上下文
   - 键值对被拆分

2. **推荐使用通用分块或递归分块**
   - 整个文件作为一个分块
   - 保持数据完整性
   - 检索效果好

3. **最佳方案：改进 CSV 格式**
   - 添加描述性的列名
   - 每行作为一个完整的问答对
   - 然后使用问答分块
